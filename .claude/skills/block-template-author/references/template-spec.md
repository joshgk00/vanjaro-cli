## File Format

Templates are stored as JSON in `artifacts/block-templates/{category}/` with kebab-case filenames.

### Required top-level fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Human-readable display name |
| `category` | string | One of the valid categories (see below) |
| `description` | string | One-line description of the UI pattern |
| `template` | object | Root section component tree |
| `styles` | array | GrapesJS style rules (usually empty `[]`) |

## Component Types

Every component object requires `type`, `classes`, and `attributes`. The `attributes` object must include an `id` field -- Vanjaro's C# renderer uses dynamic binding on `attributes.id` and crashes with a null reference if it's missing. Optional fields: `components` (children), `content` (text), `tagName`.

Use a prefix convention for template IDs: `tpl-{template-slug}-{type-letter}{n}` (e.g., `tpl-hero-s1`, `tpl-hero-g1`, `tpl-hero-h1`). IDs are regenerated by GrapesJS when the block is dropped onto a page, so the exact values don't matter -- they just need to be present and unique within the template.

| Type | Required Classes | Optional Classes | Notes |
|------|-----------------|------------------|-------|
| `section` | `vj-section` | `bg-*`, spacing | Root container. Always the top-level element. |
| `grid` | `container` OR `container-fluid` | spacing | Direct child of section only. |
| `row` | `row` | `g-{1-5}`, flex, text alignment | Direct child of grid only. |
| `column` | At least one `col-*` class | spacing, flex, text | Direct child of row. Must include `col-12` for mobile. |
| `heading` | `vj-heading`, `head-style-{1-10}` | spacing, text | Requires `tagName` (h1-h6) and `content`. |
| `text` | `vj-text`, `paragraph-style-{1-10}` | spacing, text | Requires `content` with placeholder text. |
| `button` | `btn`, one `btn-*` variant, `button-style-{1-10}` | spacing | Requires `content` for label, `tagName: "a"`, and `attributes: {role: "button", href: "#"}`. |
| `icon` | `icon-box`, `vj-icon` | `text-*`, `border-*` | Icon placeholder. |
| `image` | `image-box`, `vj-image`, `img-fluid` | spacing, sizing | Image placeholder. |
| `link` | `vj-link` | any | Wrapper that can hold other components. |
| `list` | `list-box` | spacing | Children are list items. |
| `spacer` | `spacer` | | Vertical spacing element. |
| `divider` | `vj-divider` | spacing | Horizontal rule. |
| `video` | `video-box` | sizing | Video embed container. |
| `carousel` | `carousel` | | Bootstrap 5 carousel. |
| `image-gallery` | `vj-image-gallery` | spacing | Multi-image display. |

## Nesting Rules

Strict hierarchy must be followed:

```
section (root)
  +-- grid
       +-- row
            +-- column
                 +-- leaf components (heading, text, button, icon, image, etc.)
                 +-- OR nested row (for sub-grids)
                      +-- column
                           +-- leaf components
```

### Validation checks

- `section` is always and only the root
- `grid` is a direct child of `section` (not nested elsewhere)
- `row` is a direct child of `grid` or `column` (sub-grid)
- `column` is a direct child of `row`
- Leaf components go inside `column` only
- No leaf components directly inside section, grid, or row

## Class Format

Every class entry uses GrapesJS format:

```json
{"name": "class-name", "active": false}
```

The `active: false` flag means the class IS applied. This is counterintuitive but required by GrapesJS internals.

### Rules

- No duplicate class names within a single component's `classes` array
- All entries must have both `name` and `active` fields
- `active` must be `false` (boolean, not string)

## Style Presets

Vanjaro provides numbered style presets that must be used:

| Component | Preset Pattern | Range |
|-----------|---------------|-------|
| heading | `head-style-{N}` | 1-10 |
| text | `paragraph-style-{N}` | 1-10 |
| button | `button-style-{N}` | 1-10 |

Each heading, text, or button component must include exactly one style preset class.

## Bootstrap 5 Utilities

All standard Bootstrap 5.1.0 utility classes are available:

### Spacing
`m-{0-5}`, `p-{0-5}`, `mx-auto`, `my-{0-5}`, `mt-{0-5}`, `mb-{0-5}`, `ms-{0-5}`, `me-{0-5}`, `px-{0-5}`, `py-{0-5}`, `gap-{1-5}`

### Flexbox
`d-flex`, `flex-column`, `flex-row`, `flex-wrap`, `align-items-center`, `align-items-start`, `align-items-end`, `justify-content-center`, `justify-content-between`, `justify-content-start`, `justify-content-end`

### Text
`text-center`, `text-start`, `text-end`, `text-uppercase`, `text-lowercase`, `fw-bold`, `fw-normal`, `fw-light`, `fs-{1-6}`, `lh-1`, `lh-sm`, `lh-base`, `lh-lg`

### Colors
`text-primary`, `text-secondary`, `text-success`, `text-danger`, `text-warning`, `text-info`, `text-light`, `text-dark`, `text-white`, `text-muted`
`bg-primary`, `bg-secondary`, `bg-success`, `bg-danger`, `bg-warning`, `bg-info`, `bg-light`, `bg-dark`, `bg-white`
`border-primary`, `border-secondary`, etc.

### Display
`d-none`, `d-block`, `d-flex`, `d-inline`, `d-inline-block`, `d-grid`
Responsive: `d-{sm|md|lg|xl|xxl}-{value}`

### Borders & Effects
`border`, `border-0`, `rounded`, `rounded-circle`, `rounded-pill`, `shadow`, `shadow-sm`, `shadow-lg`, `shadow-none`

### Sizing
`w-25`, `w-50`, `w-75`, `w-100`, `h-100`

### Responsive column classes
`col-{1-12}`, `col-sm-{1-12}`, `col-md-{1-12}`, `col-lg-{1-12}`, `col-xl-{1-12}`, `col-xxl-{1-12}`

### Gutter classes
`g-{0-5}`, `gx-{0-5}`, `gy-{0-5}`

## Categories

Valid template categories (case-sensitive):

- `Heroes` -- Full-width hero/banner sections
- `Content` -- General content blocks (text + image combos, feature sections)
- `Cards` -- Card-based layouts (feature cards, pricing cards, team cards)
- `Testimonials` -- Customer quotes, review sections
- `CTAs` -- Call-to-action sections
- `Lists` -- List-based layouts (feature lists, icon lists)
- `Media` -- Image galleries, video sections
- `Navigation` -- Nav bars, breadcrumbs, tab sections

## Registration Notes

The template JSON file is the **authoring format**. When registering with the CLI:

```bash
vanjaro custom-blocks create --name "Template Name" --category "Category" --file path/to/template.json
```

The CLI reads the file and extracts:
- `template` object gets wrapped in an array as `components: [template]`
- `styles` array is passed as-is

The skill produces the authoring format. The CLI handles the conversion for API registration. Do not manually wrap `template` in an array in the file.
